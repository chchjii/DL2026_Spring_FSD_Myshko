# Часть 3. Работа с AI

## 3.1. Применённые инструменты и промпты
В ходе выполнения задания использовался чат-ассистент (Claude/ChatGPT) для:
- Проектирования архитектуры базы данных и API.
- Генерации базовых шаблонов кода на Express и React.
- Настройки конфигурации Tailwind CSS и Prisma.
- Отладки ошибок, связанных с несовместимостью версий библиотек (Prisma 7, Tailwind v4).

**Примеры промптов:**
- "Помоги спроектировать схему SQLite для хранения настроек QR-кода".
- "Напиши React-компонент для Live Preview QR-кода с использованием библиотеки qrcode.react".
- "Ошибка TS1259 при импорте express в TypeScript, как исправить через tsconfig?".

## 3.2. Анализ эффективности
**Где AI помог ускорить разработку:**
- Мгновенное написание "рутинного" кода (CRUD операции, базовые стили Tailwind).
- Быстрый поиск решений для специфических ошибок TypeScript и конфигурации инструментов.
- Генерация документации (дизайн-документ и инструкции).

**Где AI ошибся:**
- AI предлагал стандартные решения для старых версий библиотек (Prisma 6 vs 7, Tailwind v3 vs v4). Ошибки обнаруживались при запуске команд `npx`, после чего требовалось уточнение промпта для адаптации кода под актуальные версии пакетов.
- Были моменты, когда предлагались избыточные функции (например, сложная проверка контрастности), от которых было решено отказаться в пользу простоты и стабильности итогового решения.

## 3.3. Выводы и рекомендации
1. AI — незаменимый инструмент для быстрого прототипирования и написания Boilerplate-кода.
2. Критически важно проверять версии предлагаемых решений, так как экосистема JS меняется быстрее, чем обновляются базы знаний моделей.
3. Рекомендую коллегам использовать AI для отладки ошибок (копировать текст ошибки в чат), это экономит часы поиска на StackOverflow.
