Часть 1. Проектирование и анализ: Генератор QR-кодов с кастомизацией

 1.1. Пользовательские сценарии (User Stories)

1. Базовый сценарий: Как маркетолог, я хочу ввести ссылку на свой сайт, выбрать фирменные цвета бренда и сразу увидеть превью QR-кода, чтобы скачать его в формате PNG для печати на флаерах.
2. Сценарий работы с историей: Как пользователь, я хочу видеть историю ранее созданных мной QR-кодов на боковой панели, чтобы иметь возможность быстро скачать их заново, если я потерял файл.
3. Продуктовое расширение (vCard): Как профессионал, я хочу сгенерировать QR-код в формате визитки (vCard), заполнив форму с именем и телефоном, чтобы мои клиенты могли добавить мой контакт в один клик после сканирования.

 1.2. Функциональные требования

Обязательные требования (Фронтенд):
- Форма ввода данных (текст или URL) с мгновенной валидацией (проверка на формат URL).
- Панель настроек внешнего вида: выбор цвета самого кода (foreground) и фона (background), настройка размера.
- Блок "Live Preview" — мгновенное обновление QR-кода при любых изменениях.
- Кнопки скачивания результата в форматах PNG и SVG.

Обязательные требования (Бэкенд):
- Обработка входящих запросов на сохранение конфигурации сгенерированного QR-кода.
- Отдача списка ранее созданных QR-кодов для конкретного пользователя (идентификация по анонимной сессии).

**Опциональные (дополнительные) функции, реализуемые в проекте:**
- Поддержка генерации QR-кодов для контактов (vCard).
- Анонимная история: генерация уникального `session_id` на клиенте (хранится в LocalStorage) для привязки истории без необходимости регистрации.

 1.3. Проектирование API

Бэкенд предоставляет REST API. Для разделения пользователей без сложной системы авторизации используется заголовок `X-Session-ID`.

1. Сохранение нового QR-кода в историю**
- Метод: `POST`
- Путь: `/api/qrcodes`
- Заголовки:** `X-Session-ID: <uuid>`
- Параметры запроса (Body):**
  ```json
  {
    "type": "url", 
    "content": "https://example.com",
    "fgColor": "#000000",
    "bgColor": "#ffffff"
  }

  Формат ответа (201 Created):
  {
  "id": 1,
  "content": "https://example.com",
  "createdAt": "2025-10-25T12:00:00Z"
}

2. Получение истории QR-кодов пользователя
Метод: GET
Путь: /api/qrcodes
Заголовки: X-Session-ID: <uuid>
Формат ответа (200 OK): Массив сохраненных объектов QR-кодов.
3. Удаление QR-кода из истории
Метод: DELETE
Путь: /api/qrcodes/:id
Заголовки: X-Session-ID: <uuid>

1.4. Модель данных
Приложение использует локальную реляционную базу данных. Храним только метаданные (настройки), так как сама картинка генерируется "на лету" на клиенте, что экономит место на сервере.
Таблица QRCode (описывается через Prisma Schema)
| Поле | Тип данных | Описание |
| :--- | :--- | :--- |
| id | Int (Auto-increment) | Уникальный идентификатор записи |
| sessionId | String | Идентификатор сессии браузера (для привязки истории) |
| type | String | Тип данных (text, url, vcard) |
| content | String | Само содержимое (ссылка или текст) |
| fgColor | String | Цвет кода (HEX) |
| bgColor | String | Цвет фона (HEX) |
| createdAt | DateTime | Дата и время создания (default: now) |

1.5. Ключевые технические решения
Фронтенд: React 18 + Vite + TypeScript.
Обоснование: Vite обеспечивает сверхбыструю сборку. React идеален для создания динамичного интерфейса (Live Preview мгновенно реагирует на изменение стейта).
Стилизация: Tailwind CSS.
Обоснование: Позволяет быстро создать минималистичный, адаптивный и красивый интерфейс без написания объемных CSS-файлов.
Генерация QR: библиотека qrcode.react.
Обоснование: Рендерит QR-коды прямо в SVG/Canvas, что позволяет легко реализовать скачивание картинок без нагрузки на сервер.
Бэкенд: Node.js + Express + TypeScript.
Обоснование: Стандартный, читаемый и надежный стек. TypeScript гарантирует совпадение типов с фронтендом.
База данных: SQLite + Prisma ORM.
Обоснование: Полностью покрывает требование тестового задания «код можно было легко запустить на Linux и Windows». База создается в виде одного файла при старте, проверяющему не придется устанавливать Docker, PostgreSQL или MySQL. Prisma делает работу с БД безопасной и удобной.
